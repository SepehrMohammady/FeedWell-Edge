Chosen Approach

Baseline to Advanced Path

  1. CL-v1 baseline: lightweight local learner + event-driven updates.
  2. CL-v2: replay + drift-aware update schedule.
  3. CL-v3: anti-collapse constraints + resource-optimized adaptation.
  4. TinyML track: compact representation for future STM32 transfer.

Why This Approach

  • Scientific control: clear baseline and ablations.
  • Deployment realism: can run in production app constraints.
  • Privacy-by-design: local processing as first principle, not add-on.
  • Transferability: method claims based on efficiency and adaptation, not RSS semantics.

User Engagement Model (3-Stage Journey)

Stage User Activity Signals Captured Available Actions
1. Feed Tab Browse article list, filter, sort, search, scroll Impression rank, filter choice, sort order, title length, image presence, language, search queries, dwell time Open preview, ambient sounds, share, save
2. Preview View article snippet, decide action Preview dwell, action type (read-in-app / browser / back) Read in app, open browser, share, save, return to feed
3. Reader Read full article, use features Scroll depth, feature usage (translate, read-aloud, notes), dwell time, completion signal Translate, read-aloud, notes, bookmark, share, save, open browser

Behavioral Insights (From User Research)

Stage 1 Dwell Factors

  • Subject interest
  • Title length
  • Language (native vs non-native)
  • Image presence
  • Feed source familiarity

Stage 3 Reading Factors

  • Character count
  • Language (native vs non-native)
  • Content complexity
  • Feature usage (TTS, translate)
  • Reading goal (completion vs skim)

Evaluation Contract (Primary Metrics)

Dimension What We Measure Why It Matters
Utility Ranking quality, engagement outcomes Confirms personalization value
Adaptation Recovery speed after drift Validates continual learning behavior
Latency Inference/update runtime on device Determines deployability in real UX
Footprint RAM use, model size, storage growth Sets transfer feasibility to constrained targets
Power Battery and thermal overhead Ensures practical long-term operation
Privacy Data locality and export surface audit Supports privacy-by-design claims
