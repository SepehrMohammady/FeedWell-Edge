Chosen Approach

Baseline to Advanced Path (Efficiency Cascade)

  1. Phase A (now): lightweight local learner + event-driven updates.
  2. Phase B (near): µNAS-inspired architecture search + BNN weight quantization (8-32x compression).
  3. Phase C (research): cross-device transfer to STM32; validate reproducibility and generalization.

Efficiency target: Phone model <5MB, inference <50ms, update <100ms.

Why This Approach

  • Scientific control: clear baseline and ablations.
  • Deployment realism: can run in production app constraints.
  • Privacy-by-design: local processing as first principle, not add-on.
  • Transferability: method claims based on efficiency and adaptation, not RSS semantics.

Monitoring export for pilot analysis Ready Use Export Telemetry Snapshot in Settings to save structured JSON metrics.
Formal experiment instrumentation Pending Add automatic latency/battery traces and aggregate reporting for multi-user studies.

How to Run Pilot Collection

  1. Enable On-Device Learning (EdgeML) in Settings.
  2. Use the app normally (browse, open, read, translate, save) to generate real interactions.
  3. Refresh Local Summary (7 days) to check event growth and top topics.
  4. Use Export Telemetry Snapshot at checkpoints (daily or session-end).
  5. Analyze snapshots for drift, topic shifts, and storage growth without exporting raw browsing histories.

User Engagement Model (3-Stage Journey)

Stage User Activity Signals Captured Available Actions
1. Feed Tab Browse article list, filter, sort, search, scroll Impression rank, filter choice, sort order, title length, image presence, language, search queries, dwell time Open preview, ambient sounds, share, save
2. Preview View article snippet, decide action Preview dwell, action type (read-in-app / browser / back) Read in app, open browser, share, save, return to feed
3. Reader Read full article, use features Scroll depth, feature usage (translate, read-aloud, notes), dwell time, completion signal Translate, read-aloud, notes, bookmark, share, save, open browser

Behavioral Insights (From User Research)

Stage 1 Dwell Factors

  • Subject interest
  • Title length

  • Feed source familiarity

Stage 3 Reading Factors

  • Character count
  • Language (native vs non-native)
  • Content complexity
  • Feature usage (TTS, translate)
  • Reading goal (completion vs skim)
